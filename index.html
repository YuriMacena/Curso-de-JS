CLIENT X SERVER

Usando um celular ou computador, você está utilizando um cliente. Você está usando um serviço fornecido por um servidor (server) neste caso, o YouTube. Em qualquer serviço que você adentrar (Google, Instagram, Facebook etc.) há a movimentação de dados para fazer esses serviços funcionarem. NO MOMENTO EM QUE VOCÊ USA UM DISPOSITIVO, ELE É UM CLIENTE. QUE PRECISA DE DADOS DE UM SERVIDOR (OU MAIS). Utilizando um navegador para acessar a internet, pesquisando e acessando uma url, irá para a infraestrutura da internet, desviando-o para um servidor. Neste servidor, está o arquivo ou algo que você esteja interessado.

Solicitando isto ao servidor, ele mandará ao client (VOCÊ) uma cópia no seu navegador. O seu navegador, apartir daí, irá trabalhar para gerar o efeito visual.

JAVASCRIPT SURGIU COMO UMA TECNOLOGIA PARA CLIENTE, PORÉM HOJE EM DIA TAMBÉM FUNCIONA EM SERVIDORES.

WEBSITE

Um website, nesse exemplo, é feito por HTML, CSS e JavaScript. O website não existiria sem os três.

Aprendendo JavaScript, inevitavelmente será necessário, também, aprofundar-se em HTML e CSS.

O JavaScript também tem o poder de modificar as áreas de HTML e CSS, sendo assim, bastante extenso.

HTML e CSS não são linguagens de programação, são tecnologias de construção de sites, por isso, nunca diga que programa em HTML ou CSS, pois ambos são utilizados apenas para o desenvolvimento. Enquanto isso, JavaScript é verdadeiramente uma linguagem de programação (HTML, CSS e JavaScript se complementam). O HTML seria responsável pela construção textual e gramatical, sendo apenas o conteúdo. O CSS seria o design para deixar a aparência mais bonita e confortável. E o JS seria responsável pela interação.

Será ensinado a forma internacional padronizada do JavaScript: ECMAScript
